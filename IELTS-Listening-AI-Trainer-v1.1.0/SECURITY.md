# Security Policy

## Reporting

Do not publish exploitable security issues in a public GitHub issue. Contact the repository owner privately with the affected version, reproduction steps, impact, and a minimal diagnostic report that contains no audio, transcript, answer, URL, or personal file-path data.

## Local data

The app is local-first. Uploaded audio, imported URL cache, models, sessions, preferences, backups, and diagnostics are excluded from source archives. Users should review diagnostic exports before sharing them.

Security fixes target the latest `1.1.x` release line. Dependency alerts and CI failures should be addressed before a release is published.
